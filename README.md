# cn-fanduel-pages

本仓库用于归档和发布多个独立的 HTML 页面。这些页面本身不绑定任何指定的域名或网站，仅作为静态页面文件的存储与分发用途。

## 项目简介

cn-fanduel-pages 是一个静态页面仓库，主要存放各种独立功能的 HTML 文件。仓库内容不限于特定主题、服务或应用场景，页面之间相互独立，便于单独引用或部署。

## 目录说明

```
├── pages/          # 存放所有 HTML 页面文件
│   ├── example1.html
│   └── example2.html
├── assets/         # 页面引用的静态资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
├── index.html      # 可选：仓库页面索引
└── README.md       # 本文件
```

- `pages/`：核心目录，所有归档的 HTML 页面均放置于此。
- `assets/`：存放页面所需的样式、脚本、媒体等资源文件，按类型分目录管理。
- `index.html`：可选索引页，用于展示仓库内页面列表或导航。

## 页面归档说明

- 所有页面均为纯前端 HTML 文件，不依赖后端服务。
- 每个页面功能独立，可单独打开或嵌入其他环境。
- 页面文件命名建议使用有意义的英文或拼音，方便识别。
- 资源文件建议使用相对路径引用，以保持仓库内结构可移植。

## 维护说明

- 新增页面请放入 `pages/` 目录，并在必要时更新索引页。
- 修改或删除页面时，请确保相关资源文件同步处理。
- 不建议在页面中硬编码外部链接或依赖第三方服务，以保持页面自包含性。
- 欢迎提交 Pull Request 贡献新页面或改进现有页面，但请确保内容符合开源社区规范。

## 使用方式

可直接通过 GitHub Pages 部署本仓库，或克隆到本地用任意静态服务器运行。每个 HTML 文件均可独立访问。
